*Исходная домашка была взята с курса "Машинное обучение-2" СБ ВШЭ-РЭШ, 2025/2026 учебный год*

Репозиторий представляет собой решение ДЗ5 по курсу ИИ в экономике (vibe-coding).

## Состав проекта

- `app.py` — Streamlit-интерфейс для инференса MNIST.
- `model.py` — класс модели и предиктор.
- `Dockerfile` — контейнеризация приложения.
- `requirements.txt` — Python-зависимости.

## Деплой в Timeweb Cloud (через контейнер)

Ниже самый прямой сценарий: загрузить репозиторий в Git и развернуть как контейнерное приложение.

### 1) Подготовьте веса модели

Приложение ожидает файл `model_weights.pth` (по умолчанию путь задаётся в UI).
Добавьте этот файл в корень проекта или измените путь в интерфейсе после запуска.

### 2) Сборка и локальная проверка Docker-образа

```bash
docker build -t mnist-app:latest .
docker run --rm -p 8501:8501 mnist-app:latest
```

Проверьте в браузере: `http://localhost:8501`.

### 3) Публикация образа в реестр

Можно использовать Docker Hub или встроенный реестр Timeweb.
Пример для Docker Hub:

```bash
docker tag mnist-app:latest <dockerhub_user>/mnist-app:latest
docker push <dockerhub_user>/mnist-app:latest
```

### 4) Создание сервиса в Timeweb Cloud

1. В панели Timeweb Cloud откройте раздел контейнерных приложений.
2. Создайте новый сервис из Docker-образа.
3. Укажите образ: `<dockerhub_user>/mnist-app:latest`.
4. Укажите порт контейнера: `8501`.
5. Пропишите переменные окружения (при необходимости).
6. Запустите деплой.

### 5) Проверка после деплоя

- Откройте выданный домен/URL сервиса.
- Убедитесь, что интерфейс Streamlit загружается.
- Проверьте предсказание на тестовом рисунке.

## Примечания

- Для быстрого старта в Docker-контекст не включаются датасет и ноутбук (см. `.dockerignore`).
- Если `model_weights.pth` отсутствует в контейнере, предсказания не заработают до указания корректного пути к весам.
